MCP Axis Status Bits

The status word that the MCP passes to the TCC upon receipt of a STATUS command is defined in data_collection.h; the version as of Guy Fawkes day 2000 is: 
struct AXIS_STAT {
	unsigned always_zero : 1;

	unsigned bump_up_ccw_sticky : 1;
	unsigned bump_dn_cw_sticky : 1;
	unsigned  : 4;
	unsigned ms_on_correction_too_large : 1;

	unsigned  : 6;
	unsigned clock_slow_signal : 1;
	unsigned clock_loss_signal : 1;
        unsigned : 5;
	unsigned stop_ok : 1;
	unsigned amp_ok : 1;
	unsigned closed_loop : 1;

	unsigned max_limit: 1;
	unsigned min_limit: 1;
	unsigned max_acc: 1;
	unsigned max_vel: 1;
	unsigned max_pos: 1;
	unsigned min_pos: 1;
	unsigned pvt_time_late : 1;
	unsigned pvt_empty : 1;
};
with meanings:
always_zero
Sign bit; always zero.
bump_up_ccw_sticky
The windscreen touched. Only valid for altitude (up) or azimuth (anticlockwise)
bump_dn_cw_sticky
The windscreen touched. Only valid for altitude (down) or azimuth (clockwise)
ms_on_correction_too_large
The correction made by an MS.ON command would have exceeded MS.MAX
clock_slow_signal
Unused
clock_loss_signal
Failed to see the GPS interrupt
stop_ok
A stop button has not been pushed.
amp_ok
The amplifier for this axis is OK
closed_loop
This axis is in closed loop
max_limit
Unused
min_limit
Unused
max_acc
Unused
max_vel
Unused
max_pos
Unused
min_pos
Unused
pvt_time_late
Unused
pvt_empty
Unused
